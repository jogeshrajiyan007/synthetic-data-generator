🏛️ Data Governance RAG API
AI-powered Data Governance framework for the insurance industry.
Quick Start
# Install dependencies
pip install -r requirements.txt

# Start the API
cd backend
uvicorn main:app --reload --port 8000
Open http://localhost:8000/docs for the interactive Swagger UI.
API Reference
POST /api/ingest/
Ingest all files currently in loading_zone/. Rebuilds vector store.
Response:
{
  "status": "success",
  "files_processed": 4,
  "files_failed": 0,
  "total_chunks": 4000,
  "total_vectors": 4000,
  "file_details": [...],
  "duration_seconds": 12.4,
  "ingested_at": "2026-04-08T10:00:00"
}
POST /api/ingest/upload
Upload files directly via multipart form. Adds to existing vector store (incremental).
POST /api/query/
Ask a governance question.
Request:
{
  "question": "What PII data is present and how should it be governed?",
  "use_rag": true,
  "k": 5
}
Response:
{
  "run_id": "uuid",
  "question": "...",
  "answer": "1. **Summary**...",
  "mode": "RAG",
  "sources_used": ["claims.csv", "customers.csv"],
  "chunks_retrieved": 4,
  "model": "gpt-4o-2024-05-13",
  "created_at": "...",
  "duration_seconds": 3.2
}
POST /api/query/batch
Run up to 20 queries in one call.
GET /api/sources/
List all ingested source files with governance metadata.
GET /api/profile/{filename}
Full statistical profiling of a CSV file — returns column-level stats as JSON.
GET /api/logs/responses
Paginated LLM response logs. Params: limit, offset.
GET /api/logs/chunks
Paginated chunk metadata logs. Params: limit, offset, source_file.
GET /api/logs/stats
Governance dashboard stats — sensitivity distribution, PII count, dept breakdown.
DELETE /api/reset/
Clear ChromaDB and SQLite. Loading zone files are preserved.
GET /health
System health — vector count, chunk count, sources loaded.

│   └── models/
