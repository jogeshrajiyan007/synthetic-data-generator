# Detailed Documentation of Profiling Notebooks, Classes, and Functions

## Overview

This documentation provides a comprehensive explanation of the `profiling_execute` and `profiling_config` notebooks, detailing the classes and functions involved, and the relationship between both notebooks.

## Profiling Execute Notebook

### Code Structure

#### Code Cells:
1. **script_cell_0**: Runs an external configuration script for the Data Profiling Framework from a specified workspace location.
2. **script_cell_1**: Reads data into a Spark DataFrame, applies filtering rules, converts date columns, and initializes a `DataProfile` object to generate various profiling DataFrames such as global, data type, unique, categorical, string, date, and numeric profiles.
3. **script_cell_2**: Converts profiling DataFrames to Pandas DataFrames, displays them, and saves them as CSV files in the Databricks File System (DBFS). It handles cases where the unique and categorical DataFrames may be empty.

### Connected Notebook
- **profiling_config**: Contains additional functions and classes used by `profiling_execute`.

## Profiling Config Notebook

### Code Structure

#### Code Cells:
1. **script_cell_0**: Imports various modules and functions from PySpark, pandas, and other libraries necessary for data manipulation and analysis.
2. **script_cell_1**: Sets the current database context to 'dmg_metadata' using SQL.
3. **script_cell_2**: Retrieves widget values for schema name, table name, and profiling tasks from Databricks.
4. **script_cell_3**: Parses profiling tasks and creates a DataFrame with column names and profiling types for further analysis.

## Functions and Classes

### Functions in `profiling_config`
1. **`readData()`**: Reads data from a specified table and joins it with metadata from 'dmg_target.metaport_ingestion_metadata', returning a Spark DataFrame and optionally a CSV file with filter rules.
2. **`filterData()`**: Applies filtering conditions to a Spark DataFrame based on rules specified in a CSV file, modifying the DataFrame accordingly.
3. **`convert_date_columns()`**: Converts date columns in a Spark DataFrame to string format and attempts to parse various date formats.

### Class in `profiling_config`
- **`DataProfile`**: Performs exploratory data analysis (EDA) on input data, including profiling data types, unique values, categorical data, and generating summaries for date and numeric columns.
  - **`__init__()`**: Initializes the class with a Spark session and defines schemas for date and numeric summary DataFrames.
  - **`global_df()`**: Generates a tuple containing the DataFrame's name, row count, and column count.
  - **`profileDataTypes()`**: Creates a list of tuples with the DataFrame's name, column names, and their data types.
  - **`unique_df()`**: Calculates unique and null counts for columns identified as requiring unique value profiling, returning a summary DataFrame.
  - **`categorical_df()`**: Profiles categorical columns by counting unique values and nulls, and checks if the unique count is below a specified threshold.
  - **`global_mask_analysis()`**: Generates masks for string columns to identify patterns and anomalies.

## Relationship Between Notebooks

The `profiling_execute` notebook relies on the `profiling_config` notebook for its functions and classes. Specifically, `profiling_execute` uses the `DataProfile` class and its methods to perform data profiling tasks. The functions in `profiling_config` are used to read, filter, and convert data, which are essential steps before profiling can be performed.

## Caveats and Limitations

- The documentation assumes familiarity with Spark, PySpark, and Databricks environments.
- The profiling tasks are dependent on the correct configuration and availability of metadata in the specified database.
- The notebooks are designed to handle specific data profiling tasks and may require customization for different datasets or profiling requirements.
