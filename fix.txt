# Data Collect – Interface Monitoring

**Last Refreshed:** February 26, 2026 &nbsp;|&nbsp; **Coverage:** 08 Feb 2026 – 14 Feb 2026

---

## 📊 What This Page Tells You

This page gives you a **detailed view of data collection activity** across all AXA entities for the selected week. It tracks how many employee records and files were received, how many passed or failed initial checks, and how collection volumes trended day by day — helping you identify which entities are driving errors or rejections.

---

## 🔢 KPI Definitions

| KPI | What it means |
|-----|--------------|
| **Employees Number** | Total distinct employee records received from all entities this week. |
| **Files Loaded** | Number of data files successfully ingested from Oracle PaaS interfaces this week. |
| **Overall Error Rate** | Percentage of all loaded records containing at least one data quality issue. Lower is better. |
| **Accepted Rows** | Records that passed initial validation checks and are cleared for downstream processing. |
| **Rejected Rows** | Records that failed one or more validation rules and are held back pending resolution. |

---

## 📈 Visual Breakdown

### Entity Performance Overview
A side-by-side bar chart showing **accepted (green) vs. rejected (red) rows per AXA entity** for the week. Use this to quickly spot which entities have high rejection volumes relative to their total load — for example, AXA UK and AXA Belgium show the largest absolute volumes, while AXA Ireland and AXA Morocco show disproportionately high rejection counts relative to their accepted rows.

### Daily – Data Collect Trends
A line chart tracking **daily record volumes** across the week (toggle between Weekly and Daily view). Use this to identify if a spike or drop in volumes is tied to a specific day's file submission — useful for diagnosing whether issues are systemic or isolated to a particular interface run.

### Rejection Rate Trends – All Entities
An area chart showing **daily rejection rate (%)** across all entities combined. This helps you see whether data quality is improving or deteriorating over the week. A rising trend (e.g. 39.72% → 69.69%) signals accumulating issues that need immediate investigation.

## 🧭 How to Navigate

- **Click "DQ Monitoring"** → Switches to the Data Quality Insights page for a deeper look at which rules are failing and why.
- **Click "Review DQ Rules"** → Opens the rule registry to inspect or validate the logic behind each check.
- **Click "Investigate Errors"** → Takes you to a row-level error log to identify and action specific failed records.
- **Click "Home"** → Returns to the main summary dashboard.
- **Click "Summary"** → Opens a consolidated weekly summary view across all metrics.
- **Click "Support"** → Raises a support request or accesses documentation.

---

> 💡 **Tip:** Use the **entity bar chart** alongside the **rejection rate trend** together — if one entity dominates rejected rows AND the rejection rate is rising daily, that entity's interface likely has a systemic issue worth escalating.
